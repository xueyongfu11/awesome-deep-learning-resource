# L2-1 经典的规划方法：新手学习课程

---

## 0. 这节课在讲什么

本节以“机器人抓取”为例，讲机器人完成一个 pick & place 任务时，经典方法大致怎样工作。

一个典型流程是：

```text
位姿估计 -> 抓取位姿计算 -> 路径规划 -> IK 求解 -> 关节空间轨迹生成 -> 执行
```

用更直白的话说：

```text
先看见物体在哪里
再决定怎么抓
再规划机械臂怎么过去
再把空间中的移动转换成关节角
最后生成平滑、可执行的运动轨迹
```

本节课重点是：

1. 规划算法是什么
2. 路径规划是什么
3. RRT / RRT-Connect / RRT* 怎么理解
4. A* / Dijkstra / PRM 怎么理解
5. 路径规划之后，为什么还需要轨迹生成

---

## 1. 先建立整体概念

### 1.1 任务规划、路径规划、轨迹生成的区别

机器人规划通常可以分成几个层次。

| 层次 | 回答的问题 | 例子 |
|---|---|---|
| 任务规划 | 先做什么，后做什么 | 先抓杯子，再放到桌上 |
| 路径规划 | 从 A 点怎么到 B 点 | 绕开障碍物到目标位置 |
| 轨迹生成 | 以什么速度、加速度运动 | 机械臂平滑地从当前关节角运动到目标关节角 |

新手最容易混淆的是“路径”和“轨迹”。

路径更像一条几何路线：

```text
A -> 中间点1 -> 中间点2 -> B
```

轨迹在路径基础上加了时间信息：

```text
t = 0s 在 A
t = 1s 到中间点1
t = 2s 到中间点2
t = 3s 到 B
```

所以：

```text
路径 = 去哪里，经过哪里
轨迹 = 什么时候到哪里，以多快速度到
```

## 2. 机器人规划里的两个空间

### 2.1 笛卡尔空间

笛卡尔空间就是我们直观看到的三维空间。

例如机器人末端夹爪的位置可以写成：

```text
x, y, z
```

如果再加上姿态，就是：

```text
x, y, z, roll, pitch, yaw
```

这里的 6 个量合起来描述“末端在三维空间中的位姿”：

| 符号 | 含义 | 新手理解 |
|---|---|---|
| x | 左右方向的位置 | 物体/夹爪在空间中横向在哪里 |
| y | 前后方向的位置 | 物体/夹爪离机器人远近在哪里 |
| z | 上下方向的位置 | 物体/夹爪有多高 |
| roll | 绕 x 轴旋转 | 像“歪头”一样左右倾斜 |
| pitch | 绕 y 轴旋转 | 像“点头/低头抬头” |
| yaw | 绕 z 轴旋转 | 像“左右摇头/转身朝向” |

所以：

```text
x, y, z 描述位置
roll, pitch, yaw 描述朝向
```

这就是常说的末端位姿。

### 2.2 构型空间 C-space

构型空间，也叫 configuration space，简称 C-space。

对机械臂来说，C-space 通常是关节角空间。

如果一个机械臂有 6 个关节，那么一个构型可以写成：

```text
q = [q1, q2, q3, q4, q5, q6]
```

每个 `qi` 是一个关节角。

所以：

```text
笛卡尔空间：描述末端夹爪在哪里
C-space：描述每个关节转到多少度
```

### 2.3 FK 和 IK

规划中经常需要在两个空间之间转换。

| 名称 | 含义 | 输入 | 输出 |
|---|---|---|---|
| FK | Forward Kinematics，正运动学 | 关节角 | 末端位姿 |
| IK | Inverse Kinematics，逆运动学 | 末端位姿 | 关节角 |

简单理解：

```text
FK：给我关节角，我算夹爪在哪里
IK：给我夹爪目标位置，我算关节该怎么转
```

## 3. 路径规划到底要解决什么问题

### 3.1 最简单情况

如果没有障碍物，从 A 点到 B 点可以直接走直线。

```text
A ---------------- B
```

### 3.2 真实情况

现实中有障碍物，直线可能会碰撞。

```text
A ---- 障碍物 ---- B
```

这时机器人要找一条可行路径：

```text
A -> 绕开障碍物 -> B
```

路径规划要回答：

1. 有没有一条不碰撞的路
2. 如果有，怎么找到它
3. 找到的路是否足够短、足够平滑、足够安全

## 4. 基于采样的路径规划

这一部分重点讲 sampling-based planning，也就是基于采样的规划。

### 4.1 为什么要采样

机械臂的空间通常是高维的。

例如 6 自由度机械臂的 C-space 是 6 维空间。高维空间中直接精确搜索非常困难，所以经典方法常用“随机采样”。

核心思想是：

```text
不试图穷举整个空间
而是在空间中随机撒点
再把可行的点连接起来
最终找到从起点到终点的路线
```

### 4.2 采样规划的三个基本操作

| 操作 | 作用 |
|---|---|
| Sampling | 随机采样一个点 |
| Local planning | 尝试把两个点用一小段路径连接起来 |
| Collision detection | 检查连接路径是否碰撞 |

整个采样规划反复做：

```text
采样 -> 连接 -> 碰撞检测 -> 保留可行路径
```

## 5. RRT：快速随机树

RRT 全称是 Rapidly-exploring Random Tree。

它是一种基于树结构的 single-query 规划方法。

### 5.1 single-query 是什么

single-query 指的是：

```text
针对当前这一次起点和终点，临时规划一条路
```

如果环境变化很快，或者每次任务的起点终点差异很大，RRT 这类方法比较合适。

### 5.2 RRT 的直觉

RRT 就像从起点长出一棵树。

每次做：

1. 在自由空间里随机采样一个点 `x_rand`
2. 在已有树中找离它最近的点 `x_near`
3. 从 `x_near` 朝 `x_rand` 方向走一小步，生成 `x_new`
4. 检查这段小路径是否碰撞
5. 如果不碰撞，就把 `x_new` 加入树
6. 重复，直到树接近目标

可以写成伪代码：

```text
Tree = {start}

repeat:
    x_rand = random_sample()
    x_near = nearest_node(Tree, x_rand)
    x_new = steer(x_near, x_rand)

    if collision_free(x_near, x_new):
        Tree.add(x_new)
        Tree.add_edge(x_near, x_new)

    if x_new reaches goal:
        return path
```

### 5.3 RRT 的优点和缺点

| 优点 | 缺点 |
|---|---|
| 简单，容易实现 | 找到的路径不一定短 |
| 适合高维空间 | 路径可能弯弯曲曲 |
| 能较快找到可行解 | 随机性强，每次结果可能不同 |

新手要记住：

```text
RRT 主要追求先找到一条可行路径，不保证一开始就是最优路径。
```

## 6. RRT-Connect：双向生长的 RRT

RRT-Connect 是 RRT 的实用增强版本。

### 6.1 核心思想

普通 RRT 通常从起点长树。

RRT-Connect 同时从起点和终点长两棵树：

```text
start tree  ----->     <-----  goal tree
```

如果两棵树连接上，就找到路径。

### 6.2 为什么更快

因为两边同时探索，中间相遇的机会更大。

对新手可以这样理解：

```text
一个人从起点找终点，可能慢
两个人分别从起点和终点出发，中间碰头，通常更快
```

### 6.3 什么时候用

RRT-Connect 简单、有效，实际机器人路径规划中很常见。

它适合：

1. 需要快速找到可行路径
2. 环境不是特别简单
3. 不强求第一次规划就是最优路径

## 7. RRT*：会逐渐优化路径的 RRT

RRT* 是 RRT 的改进版本。

### 7.1 RRT* 想解决什么问题

RRT 能找到可行路径，但路径质量可能不好。

RRT* 增加了优化机制，让路径随着采样次数增加逐渐变好。

RRT* 有两个重要性质：

| 性质 | 含义 |
|---|---|
| Probabilistic Completeness | 只要解存在，采样足够多时，找到解的概率趋近于 1 |
| Asymptotic Optimality | 采样足够多时，路径会逐渐接近最优 |

### 7.2 RRT* 的关键：考虑历史代价

普通 RRT 更关注：

```text
哪个节点离随机点最近
```

RRT* 不只看局部距离，还看从起点走到当前节点的总代价。

它比较的是：

```text
起点 -> 某个候选父节点 -> 新节点
```

这条完整路径的 cost。

### 7.3 RRT* 的关键：rewire

rewire 可以理解为“重新接线”。

如果新加入的节点能让附近节点用更短路径连接到起点，RRT* 会修改树结构。

直觉如下：

```text
旧路径：start -> A -> B
新路径：start -> C -> B

如果新路径更短，就让 B 改接到 C
```

### 7.4 RRT 和 RRT* 的区别

| 对比 | RRT | RRT* |
|---|---|---|
| 主要目标 | 快速找到可行路径 | 找到并逐渐优化路径 |
| 是否优化 | 弱 | 强 |
| 计算量 | 较小 | 较大 |
| 路径质量 | 可能较差 | 通常更好 |

新手要记住：

```text
RRT 是先找到路。
RRT* 是找到路以后，还会不断把路变好。
```

## 8. 图搜索：Dijkstra 和 A*

接下来从树结构方法转到图结构方法。

### 8.1 图是什么

图由节点和边组成。

```text
node：一个位置或构型
edge：两个节点之间的连接路径
```

路径规划中，如果已经有一张图，那么问题就变成：

```text
从起点节点到目标节点，找一条代价最小的路径
```

### 8.2 Dijkstra

Dijkstra 是经典最短路径算法。

它的核心思想：

```text
从起点开始，逐步扩展当前已知代价最小的节点
直到找到目标
```

它只看已经走过的真实代价。

可以理解为：

```text
我不知道目标在哪个方向更好，所以稳扎稳打，把近的地方都探索清楚。
```

### 8.3 A*

A* 在 Dijkstra 的基础上加入启发式信息。

它常用的评价函数是：

```text
f(n) = g(n) + h(n)
```

其中：

| 符号 | 含义 |
|---|---|
| `g(n)` | 从起点走到当前节点的实际代价 |
| `h(n)` | 从当前节点到目标的估计代价 |
| `f(n)` | 综合评价 |

新手理解：

```text
Dijkstra：只看已经花了多少代价
A*：既看已经花了多少，也估计离目标还要多少
```

如果启发函数设计得好，A* 通常比 Dijkstra 更快。

## 9. PRM：概率路图

PRM 全称是 Probabilistic Roadmap。

它是基于图结构的 multi-query 方法。

### 9.1 multi-query 是什么

multi-query 指的是：

```text
先构建一张可重复使用的地图
之后不同起点和终点都可以在这张图上查询路径
```

如果环境比较静态，PRM 很有价值。

### 9.2 PRM 的两个阶段

PRM 分为：

1. 初始化阶段
2. 查询阶段

初始化阶段：

```text
随机采样很多节点
检查节点是否在自由空间
尝试连接附近节点
用碰撞检测保留可行边
形成 roadmap
```

查询阶段：

```text
把起点连接到 roadmap
把目标点连接到 roadmap
在图上用 A* 或 Dijkstra 找最短路
```

### 9.3 PRM 的优点和缺点

| 优点 | 缺点 |
|---|---|
| roadmap 可复用 | 初始建图成本较高 |
| 适合静态环境 | 环境变化后图可能失效 |
| 可结合 A* 等图搜索算法 | 高维空间中采样和连接仍然困难 |

新手要记住：

```text
RRT 像临时找路。
PRM 像先修好道路网，以后反复查路线。
```

---

## 10. RRT、RRT*、PRM 怎么选

| 场景 | 更适合的方法 |
|---|---|
| 环境变化快 | RRT / RRT-Connect |
| 只需要快速可行解 | RRT-Connect |
| 希望路径逐渐优化 | RRT* |
| 环境静态，需要多次查询 | PRM |
| 已有拓扑图，只需要查最短路 | A* / Dijkstra |

实际中方法可以混合使用。

例如：

```text
PRM 提供粗粒度 roadmap
RRT 在局部复杂区域继续搜索
```

## 11. 路径规划之后：轨迹生成

### 11.1 为什么有了路径还不够

路径只告诉机器人经过哪些点。

但机器人执行时还需要知道：

1. 每个时刻在哪
2. 速度是多少
3. 加速度是多少
4. 是否超过电机限制
5. 运动是否平滑

所以路径规划后还要做轨迹生成。

### 11.2 Point-to-point trajectory

点到点轨迹是最基础的问题：

```text
从起点运动到终点
```

常见方法：

1. Polynomial，多项式轨迹
2. 梯形速度规划
3. S 曲线规划

### 11.3 多项式轨迹

多项式轨迹用一个多项式描述位置随时间变化。

例如：

```text
q(t) = a0 + a1 t + a2 t^2 + a3 t^3
```

通过设置起点和终点的位置、速度等约束，可以求出系数。

新手只需要先理解：

```text
多项式轨迹的目标是让运动连续、平滑。
```

### 11.4 梯形速度规划

梯形速度规划通常包括三个阶段：

```text
加速 -> 匀速 -> 减速
```

速度曲线像梯形，所以叫梯形规划。

它适合简单的点到点运动。

### 11.5 S 曲线规划

S 曲线规划比梯形规划更平滑。

它不仅考虑速度和加速度，还会考虑加加速度，也就是 jerk。

直觉上：

```text
梯形规划：速度变化比较直接
S 曲线规划：速度变化更柔和
```

对真实机器人来说，S 曲线通常能减少冲击和振动。

## 12. Via-point 和样条曲线

### 12.1 Via-point 是什么

Via-point 是中间经过点。

例如：

```text
起点 -> via point 1 -> via point 2 -> 终点
```

这些点可能来自路径规划。

轨迹生成要让机器人平滑地经过这些点。

### 12.2 多个 Via-point 的问题

难点是：

1. 每段之间速度要连续
2. 加速度不能突变太大
3. 整体运动不能抖动
4. 要满足机器人关节限制

### 12.3 样条曲线

样条曲线用于连接多个点，并保持平滑。

这里要注意：

1. B-spline
2. Polynomial spline
3. Cubic spline
4. Natural spline
5. Not-a-knot spline

新手先记住：

```text
样条曲线适合把多个中间点连接成平滑轨迹。
```

```text
机器人抓取任务
    |
    v

    |
    v

    |
    +-- RRT：快速找可行路径
    +-- RRT-Connect：双向找路，更实用
    +-- RRT*：边找边优化

轨迹生成
    |
    +-- 多项式轨迹
    +-- 梯形速度规划
    +-- S 曲线规划
    +-- 样条曲线
    |
    v
机器人执行
```
