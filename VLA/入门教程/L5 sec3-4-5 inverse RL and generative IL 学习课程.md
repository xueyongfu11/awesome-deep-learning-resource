# L5 sec3-4-5 Inverse RL and Generative IL：新手学习课程

---

## 0. 这节课在讲什么

本节继续讲模仿学习，也就是 Imitation Learning，简称 IL。

前面已经学习过：

```text
Behavior Cloning：直接从专家状态-动作数据中学习策略
Interactive IL：训练过程中让专家持续参与，修正学习策略的错误
```

本节重点是另外两类思路：

```text
Inverse Reinforcement Learning：不直接学动作，而是反推出专家在优化什么奖励
Generative IL：用生成模型学习专家轨迹、动作或策略分布
```

用一句话概括：

```text
BC 学专家怎么做
IRL 学专家为什么这么做
Generative IL 学专家行为背后的数据分布
```

本节可以分成三部分：

1. Inverse RL：逆强化学习
2. IL with Sequence Information：带序列信息的模仿学习
3. Generative Model for IL：GAN、VAE、Diffusion 在模仿学习中的用法

## 1. 先复习 MDP

理解 IRL 之前，需要先回到强化学习里的 MDP。

MDP 是 Markov Decision Process，中文通常叫马尔可夫决策过程。

一个 MDP 通常包含：

| 符号 | 含义 | 直观解释 |
|---|---|---|
| `s` | state | 当前状态 |
| `a` | action | 当前动作 |
| `p` | transition / dynamics | 执行动作后环境怎么变化 |
| `r` | reward | 当前行为得到多少奖励 |
| `o` | observation | 观测，有时不等于完整状态 |
| `π` | policy | 策略，即状态到动作的映射 |
| rollout | rollout | 从初始状态开始，连续执行策略得到一条轨迹 |

可以写成：

```text
MDP = (S, A, P, R)
```

其中：

```text
S：状态空间
A：动作空间
P：状态转移概率
R：奖励函数
```

### 1.1 Markov property 是什么

马尔可夫性质的意思是：

```text
当前状态已经包含做决策需要的信息
未来只依赖当前状态和当前动作，不再依赖更早的历史
```

形式上可以理解为：

```text
P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, ...)
= P(s_{t+1} | s_t, a_t)
```

直白地说：

```text
如果当前状态描述得足够完整，就不用再翻历史账。
```

例如在棋类游戏中，当前棋盘局面基本决定了下一步可选动作，不需要知道这些棋子之前是怎么走到这里的。

但在真实机器人任务里，当前观测可能不完整，所以历史信息有时仍然很重要。这也是后面 sequence information 这一部分要解决的问题。

## 2. 从 BC、RL 到 IRL

先把三类学习范式放在一起比较。

### 2.1 Behavior Cloning

BC 的输入是专家轨迹：

```text
专家轨迹 D = {(s, a)}
```

目标是直接学习：

```text
π(a | s)
```

也就是：

```text
看到专家在状态 s 做了动作 a
模型以后遇到类似状态，也输出类似动作
```

BC 本质上很像监督学习。

缺点是：如果模型进入了专家数据中很少出现的状态，就容易持续犯错，也就是 covariate shift。

### 2.2 Reinforcement Learning

RL 的输入通常是奖励函数：

```text
R(s, a)
```

目标是学习一个策略：

```text
π*
```

让长期累计奖励最大：

```text
max E[sum_t r_t]
```

也就是说：

```text
先定义什么是好，再让智能体自己探索怎么做好。
```

RL 的难点之一是：奖励函数很难手工设计。

### 2.3 Inverse Reinforcement Learning

IRL 的输入是专家轨迹：

```text
D = {τ_1, τ_2, ..., τ_N}
```

但它不直接学习动作，而是学习奖励函数：

```text
R(s, a)
```

核心假设是：

```text
专家之所以这样行动，是因为专家在某个未知奖励函数下接近最优。
```

所以 IRL 的目标是：

```text
从专家行为中反推出专家正在优化的奖励函数
```

再用这个奖励函数训练策略。

可以把三者关系记成：

```text
BC：专家轨迹 -> 策略
RL：奖励函数 -> 策略
IRL：专家轨迹 -> 奖励函数 -> 策略
```

---

## 3. 为什么需要 Inverse RL

IRL 的动机主要来自两个问题。

### 3.1 奖励函数很难手写

在简单任务中，奖励可以写得很直接。

例如：

```text
到达目标 +1
碰撞 -1
每走一步 -0.01
```

但机器人任务通常没这么简单。

比如自动驾驶中，“好驾驶”可能同时包含：

- 不撞车
- 不压线
- 不急刹
- 保持合理车距
- 让乘客舒适
- 遵守交通规则
- 在必要时果断换道

这些目标之间还会冲突。

例如：

```text
太保守 -> 安全但效率低
太激进 -> 效率高但风险大
```

所以真实任务中的 reward 往往不是一个简单公式。

### 3.2 专家示范更容易获得

有些任务中，人类很难准确写出奖励函数，但可以展示“怎么做”。

例如：

```text
人类司机开一段路
机械臂操作者完成一次插孔
医生演示一个操作流程
```

IRL 的想法是：

```text
既然专家行为已经体现了偏好，那就从行为里学习偏好。
```

---

## 4. Reward Learning：奖励函数怎么表示

一个常见假设是：

```text
奖励函数是特征的函数
```

最常见的简化形式是线性组合：

```text
Rθ(s) = θ^T φ(s)
```

或者考虑动作：

```text
Rθ(s, a) = θ^T φ(s, a)
```

其中：

```text
φ(s)：从状态中提取的特征
θ：每个特征的权重
Rθ：由参数 θ 决定的奖励函数
```

### 4.1 一个直观例子

假设自动驾驶里有这些特征：

| 特征 | 含义 |
|---|---|
| `φ1` | 距离前车是否太近 |
| `φ2` | 是否偏离车道中心 |
| `φ3` | 加速度是否太大 |
| `φ4` | 是否接近目标路线 |

奖励可以写成：

```text
R(s) = θ1 φ1(s) + θ2 φ2(s) + θ3 φ3(s) + θ4 φ4(s)
```

如果学到：

```text
θ1 很负
θ2 很负
θ3 略负
θ4 很正
```

说明专家行为中体现出的偏好可能是：

```text
非常不喜欢追尾风险
非常不喜欢偏离车道
稍微不喜欢急加速急减速
喜欢朝目标路线前进
```

### 4.2 奖励是有歧义的

特别强调：

```text
Reward is ambiguous.
```

也就是说，同一组专家行为可能对应多个奖励函数。

例如专家总是走中间车道，可能有很多解释：

```text
喜欢离路边远
喜欢保持车道中心
不喜欢变道
中间车道刚好最短
```

这些 reward 都可能解释同一组行为。

所以 IRL 不是在找唯一真相，而是在找一个能够合理解释专家行为、并能用于训练策略的奖励函数。

## 5. Max Entropy IRL

Max Entropy IRL 是一种重要的 IRL 方法。

它的目标是：

```text
找到一个轨迹分布，
在期望上匹配专家的特征统计，
同时让分布的熵最大。
```

拆开理解就是两件事：

1. 模型生成的轨迹应该和专家轨迹在关键特征上相似
2. 在满足相似的前提下，不要做过多额外假设

### 5.1 什么是 feature count

一条轨迹可以写成：

```text
τ = (s0, a0, s1, a1, ..., sT)
```

这条轨迹的特征累计可以写成：

```text
φ(τ) = sum_t φ(s_t, a_t)
```

专家数据中的平均特征累计是：

```text
E_D[φ(τ)]
```

IRL 希望模型策略产生的轨迹也有类似的特征累计：

```text
E_π[φ(τ)] ≈ E_D[φ(τ)]
```

直白地说：

```text
专家经常保持车距，模型也应该经常保持车距
专家很少急刹，模型也应该很少急刹
专家会朝目标前进，模型也应该朝目标前进
```

### 5.2 为什么要最大熵

如果只要求匹配特征统计，可能会得到过于确定、过于武断的解。

最大熵原则的思想是：

```text
在已知约束之外，不额外假设模型偏好。
```

也就是说：

```text
只要行为和专家在关键特征上匹配，
就不要强行规定模型必须完全复现某一条具体轨迹。
```

这对模仿学习很重要，因为专家完成同一个任务可能有多种合理方式。

例如从 A 到 B，专家可能走左边，也可能走右边。只要都安全、平滑、到达目标，就不应该只允许一种路线。

### 5.3 Max Entropy IRL 的流程

的流程可以整理为：

```text
输入：专家轨迹 D
初始化：奖励函数 Rθ(s, a)

循环：
1. 用当前奖励函数 Rθ 训练或求解最优策略 πθ
2. 用策略 πθ rollout，估计状态访问频率
3. 计算模型轨迹和专家轨迹的特征统计差异
4. 根据差异更新 θ

输出：学到的奖励函数 Rθ，以及可由该奖励训练出的策略 πθ
```

其中最关键的是梯度更新。

直观上：

```text
如果专家经常访问某类状态，而当前策略很少访问，
就提高这类状态相关特征的奖励权重。

如果当前策略经常访问某类状态，而专家很少访问，
就降低这类状态相关特征的奖励权重。
```

可以记成：

```text
梯度方向 ≈ 专家特征期望 - 当前策略特征期望
```

## 6. IRL 的高层训练闭环

IRL 不是只训练一次 reward 就结束，它通常是一个交替优化过程。

高层流程是：

```text
专家示范
  -> 学习奖励函数
  -> 用奖励函数训练策略
  -> 比较学习策略和专家策略
  -> 更新奖励函数
  -> 重复
```

可以画成：

```text
Expert Trajectories
        |
        v
Learn Reward Function
        |
        v
Train Policy with RL
        |
        v
Compare with Expert
        |
        v
Update Reward
```

### 6.1 IRL 和 Interactive IL 的区别

Interactive IL 依赖专家在训练过程中提供反馈：

```text
当前策略跑到一个状态 -> 问专家这个状态该怎么做
```

IRL 更关注从专家轨迹中解释奖励：

```text
专家为什么会这样做 -> 哪些状态和动作应该被奖励
```

### 6.2 IRL 和 RL 的区别

RL 需要人为给 reward：

```text
人写 reward -> 智能体学习策略
```

IRL 反过来：

```text
专家轨迹 -> 学 reward -> 再学习策略
```

所以 IRL 可以看成 RL 的前置步骤。

## 7. BC、Interactive IL、IRL 小结

这三类方法可以按“专家信息如何使用”来理解。

| 方法 | 输入 | 学什么 | 优点 | 主要问题 |
|---|---|---|---|---|
| BC | 离线专家状态-动作对 | 策略 | 简单，像监督学习 | 分布偏移，错误会累积 |
| Interactive IL | rollout 中的专家反馈 | 策略 | 能纠正模型自己访问到的状态 | 需要专家持续参与 |
| IRL | 专家轨迹 | 奖励函数，再学策略 | 能学习任务偏好，可迁移到 RL | 奖励有歧义，训练复杂 |

更直观地记：

```text
BC：照着专家动作学
Interactive IL：边犯错边问专家
IRL：从专家行为里反推目标
```

## 8. IL with Sequence Information

课程后半部分进入 “Others”，先讲带序列信息的 IL。

传统 BC 常常把每个样本看成独立的：

```text
(当前观测 o_t) -> (当前动作 a_t)
```

但很多任务中，历史信息很关键。

例如自动驾驶：

```text
只看当前一帧，很难判断前车是不是正在急刹
只看当前位置，很难判断行人是不是准备横穿
只看当前车道，很难判断旁边车辆是不是正在并线
```

所以策略可能需要使用一段历史：

```text
(o_{t-k}, ..., o_t) -> (a_t 或未来轨迹)
```

也可能直接预测未来一段时间：

```text
(历史观测) -> (未来 H 步轨迹)
```

### 8.1 用历史预测未来

自动驾驶中也会涉及 motion prediction 和 ML planning。

一个典型思路是：

```text
输入：过去若干帧的场景、车辆状态、地图信息
输出：未来一段时间的轨迹
```

这种方式不仅预测一个动作，而是预测一段未来行为。

例如：

```text
未来 3 秒内车辆应该如何转向、加速、减速
```

### 8.2 CCIL

CCIL 指 Context-conditioned Imitation Learning for Urban Driving。

它的核心直觉是：

```text
用上下文和历史信息预测未来轨迹，
再在评估时用最优控制进一步改善轨迹。
```

可以理解为两个阶段：

```text
学习模型：从数据中学会生成合理轨迹
控制优化：在执行时让轨迹更满足动力学和约束
```

这说明在真实机器人和自动驾驶任务里，学习方法常常会和传统控制方法结合。

## 9. Transformer 和序列建模

课程补充要点：

```text
Trajectory Transformer
Decision Transformer
```

它们都把强化学习或模仿学习问题转成序列建模问题。

### 9.1 Trajectory Transformer

Trajectory Transformer 的想法是：

```text
把轨迹看成一个长序列，
用 Transformer 学习状态、动作、奖励之间的模式。
```

原本 RL 里的数据是：

```text
s0, a0, r0, s1, a1, r1, ...
```

Transformer 可以像处理语言序列一样处理它：

```text
token1, token2, token3, ...
```

所以标题 “Offline Reinforcement Learning as One Big Sequence Modeling Problem” 的意思就是：

```text
把离线强化学习看成一个大的序列建模问题。
```

### 9.2 Decision Transformer

Decision Transformer 也用 Transformer，但它的关键输入是 return-to-go。

可以理解为：

```text
告诉模型我想得到多高的回报，
模型根据这个目标生成对应动作。
```

输入序列大致包含：

```text
目标回报、状态、动作、目标回报、状态、动作、...
```

这样策略学习就变成：

```text
给定目标回报和历史轨迹，预测下一步动作
```

这类方法体现了一个趋势：

```text
机器人决策不一定只能写成传统 RL 优化问题，
也可以写成大规模序列预测问题。
```

## 10. Generative Model for IL

生成模型的目标是学习数据分布。

对 generative model 的定义可以整理为：

```text
学习一个与数据匹配的联合分布
然后从学到的分布中采样，生成新的数据
```

在模仿学习中，数据可以是：

```text
动作
轨迹
未来轨迹
状态-动作序列
机器人关节序列
末端执行器轨迹
```

所以生成模型用于 IL 时，核心问题是：

```text
如何生成像专家一样的行为样本
```

三类生成模型包括：

```text
GAN -> GAIL
VAE -> ACT 等方法
Diffusion Model -> Diffusion Policy / Diffuser
```

---

## 11. GAIL：Generative Adversarial Imitation Learning

GAIL 是 GAN 思想在模仿学习里的代表方法。

GAN 包含两个角色：

```text
Generator：生成器
Discriminator：判别器
```

在普通 GAN 中：

```text
生成器生成假图片
判别器判断图片是真的还是生成的
生成器努力骗过判别器
```

在 GAIL 中：

```text
策略 π 相当于生成器
判别器 D 判断一段状态-动作来自专家还是来自当前策略
策略努力生成让判别器分不出来的行为
```

### 11.1 GAIL 的训练流程

的 GAIL Procedure 可以整理为：

```text
1. 用当前策略采样轨迹
2. 更新判别器：
   让它区分专家轨迹和当前策略轨迹
3. 更新学生策略：
   让它生成更像专家的轨迹，使判别器更难区分
4. 重复
```

直观类比：

```text
判别器：这段行为像不像专家？
策略：我要做得更像专家，让你判断不出来。
```

### 11.2 GAIL 和 IRL 的关系

GAIL 可以看成一种不显式手写 reward 的模仿学习方法。

但它内部其实会把判别器信号转成类似 reward 的训练信号。

也就是说：

```text
判别器越觉得当前行为像专家，策略越应该被奖励。
```

这和 IRL 的思想有联系：

```text
都不是简单做监督学习
都关心策略产生的轨迹分布是否像专家
```

## 12. VAE：Variational Autoencoder

对 VAE 的结构总结为：

```text
Encoder -> Latent Vector / Distribution -> Decoder
```

也就是：

```text
输入数据 x
编码器把 x 压缩到潜变量 z 的分布
从 z 中采样
解码器根据 z 重建或生成 x'
```

### 12.1 为什么是分布

普通 autoencoder 学到的是一个确定的向量。

VAE 学到的是潜变量分布，例如：

```text
z ~ N(μ, σ)
```

这样就可以采样：

```text
从学到的分布中采样 z
再用 decoder 生成新的样本
```

### 12.2 VAE 在 IL 中怎么用

在模仿学习中，VAE 可以用来生成：

```text
未来动作序列
机器人轨迹
抓取姿态
多模态行为
```

多模态行为很重要。

例如同一个状态下，专家可能有多种合理动作：

```text
从左边绕过去
从右边绕过去
先推一下再抓
直接抓
```

如果用普通回归模型，可能会把多种动作平均起来，得到一个反而不合理的动作。

VAE 可以通过潜变量表达不同模式：

```text
z1 -> 左绕
z2 -> 右绕
z3 -> 直接抓
```

### 12.3 CVAE

CVAE 是 Conditional VAE，也就是条件 VAE。

普通 VAE：

```text
z -> x
```

CVAE：

```text
condition c + z -> x
```

在机器人任务中，条件可以是：

```text
当前图像
当前机器人状态
目标语言指令
物体点云
任务类别
```

所以 CVAE 更适合条件生成：

```text
给定当前场景，生成一条合理动作序列
```

## 13. ACT：VAE + Transformer + IL

ACT 可以理解为：

```text
Action Chunking with Transformers
```

它结合了：

```text
VAE：建模动作序列的潜在分布
Transformer：处理长序列和上下文
IL：从专家示范中学习机器人动作
```

### 13.1 ACT 的核心问题

机器人控制如果每一步都预测一个很小的动作，可能会有两个问题：

```text
短视：每一步只看当前动作，缺乏长期结构
抖动：动作逐步预测容易不平滑
```

ACT 的思想是预测 action chunk：

```text
一次预测未来一段动作序列
```

例如：

```text
不是只预测 a_t
而是预测 [a_t, a_{t+1}, ..., a_{t+H}]
```

这样模型更容易学到连贯动作。

### 13.2 为什么结合 VAE

同一个场景下可能有多种合理动作序列。

VAE 的潜变量可以表达这种多样性：

```text
同一个观察输入 + 不同 z
-> 不同但合理的动作 chunk
```

### 13.3 为什么结合 Transformer

Transformer 适合处理序列关系。

在 ACT 中，它可以融合：

```text
视觉特征
机器人关节状态
历史动作
未来动作 chunk
```

它的优势是能建模较长时间范围内的依赖，而不是只看单步动作。

## 14. Diffusion Model + IL

Diffusion model 是近几年很重要的生成模型。

给出的核心过程是：

```text
Forward process：加噪声
Backward process：去噪声
```

更具体地说：

```text
前向过程：把真实数据逐步加噪，最后变成接近纯高斯噪声
反向过程：训练神经网络，从噪声一步步还原出数据
```

### 14.1 用在动作生成中

如果把动作序列看成数据：

```text
action sequence = [a_t, a_{t+1}, ..., a_{t+H}]
```

扩散模型就可以学习：

```text
从随机噪声生成一段像专家的动作序列
```

在执行时：

```text
输入当前观测
从噪声开始逐步去噪
输出未来动作序列
```

### 14.2 Diffusion Policy

Diffusion Policy 的核心想法是：

```text
用扩散模型生成机器人动作序列，
把 visuomotor policy 学习转成条件动作生成问题。
```

条件可以包括：

```text
当前图像
机器人 proprioception
任务状态
历史观测
```

输出通常是一段动作：

```text
未来 H 步动作
```

这类方法适合处理多模态动作分布。

例如同一个场景下：

```text
可以先从左侧接近
也可以从右侧接近
可以先调整姿态
也可以直接执行
```

扩散模型比简单回归更容易表达这种多个合理答案。

### 14.3 Diffuser

Diffuser 指 Planning with Diffusion for Flexible Behavior Synthesis。

它的思路可以理解为：

```text
把规划问题也看成轨迹生成问题。
```

也就是：

```text
不是一步步搜索轨迹，
而是用扩散模型生成满足条件的轨迹。
```

条件可以是：

```text
起点
终点
任务目标
奖励约束
环境约束
```

这说明扩散模型不仅可以做动作模仿，也可以和规划结合。

## 15. 本节核心关系图

可以把本节内容组织成下面这张图：

```text
Imitation Learning
|
|-- Behavior Cloning
|   |-- 直接监督学习
|   |-- 输入专家状态-动作对
|
|-- Interactive IL
|   |-- DAgger / HG-DAgger
|   |-- 训练时持续询问专家
|
|-- Inverse RL
|   |-- 从专家轨迹反推奖励函数
|   |-- 再用 RL 训练策略
|   |-- Max Entropy IRL
|
|-- Sequence-based IL
|   |-- 使用历史信息
|   |-- 预测未来轨迹
|   |-- Trajectory Transformer / Decision Transformer
|
|-- Generative IL
    |-- GAIL：对抗学习专家轨迹分布
    |-- VAE / CVAE：潜变量生成多模态行为
    |-- ACT：VAE + Transformer + 动作块预测
    |-- Diffusion Policy：扩散模型生成动作序列
    |-- Diffuser：扩散模型生成轨迹或规划结果
```
