## 0. 这节课在讲什么

本节课把“神经网络怎么用于图像”串起来，重点不是推导复杂数学，而是建立工程视角。

```text
神经网络基础 -> 2D 图像特征处理 -> 3D 点云特征处理
```

## 1. 神经网络和 2D 图像处理简要总结

神经网络训练的核心目标是让 `loss` 变小。`loss` 衡量预测结果和真实标签之间的差距，训练时通过反向传播和优化器不断更新参数。

2D 图像常见输入形式是 `[Batch, Channel, H, W]`。常见层包括 `Conv2d`、`Linear`、`Pooling`、`Activation` 和 `Normalization`：卷积提特征，全连接做映射，池化降尺寸，激活函数引入非线性，归一化让训练更稳定。

常见架构可以先这样记：`CNN` 适合提取图像局部特征，`Residual block` 让深层网络更容易训练，`U-Net` 常用于分割，`ViT` 把图像切成 patch 后用 Transformer 建模。

2D 图像任务主要有三类：分类判断“图里是什么”，检测判断“有什么、在哪里”，分割判断“每个像素是什么”。整体流程可以概括为：训练时学习参数，推理时输出预测，部署时把训练好的模型放到实际设备上运行。

## 2. 3D Point Cloud Feature：点云特征

### 2.1 什么是点云

点云是由很多 3D 点组成的数据。

每个点通常至少有三个坐标：

```text
(x, y, z)
```

如果有 N 个点，可以写成：

```text
N * 3
```

PointNet 的输入通常是：

```text
Input: N * 3 3D points
```

点云常来自：

```text
深度相机
激光雷达 LiDAR
RGB-D 相机
三维扫描
```

### 2.2 2D 图像和 3D 点云的区别

| 数据 | 结构 | 常见处理方式 |
|---|---|---|
| 2D 图像 | 规则网格 | Conv2d、CNN、ViT |
| 3D 点云 | 无规则点集合 | PointNet、PointNet++、点云 Transformer |

图像像素有固定邻居关系：

```text
左边、右边、上边、下边
```

点云则更像一堆散点：

```text
点的顺序不一定有意义
点的密度可能不均匀
局部邻域需要额外计算
```

这就是点云处理比普通图像更特殊的地方。

## 3. PointNet

### 3.1 PointNet 解决什么问题

PointNet 是经典点云网络。它直接处理点集合。

PointNet 的特点可以概括为：

```text
Input: N * 3 3D points
Output: Global 1D feature
Simple structure, mostly conv1d
```

### 3.2 Global 1D feature 是什么

假设输入是 N 个点：

```text
点1, 点2, 点3, ..., 点N
```

PointNet 会先对每个点提取特征，然后用类似 max pooling 的操作聚合成一个全局特征。

```text
N 个点的特征 -> 聚合 -> 一个全局向量
```

这个全局向量可以用于：

```text
点云分类
场景理解
下游任务的特征输入
```

### 3.3 为什么 PointNet 常用 conv1d

在点云中，每个点可以看成一个“小向量”。

`conv1d` 可以对每个点共享同一组 MLP/卷积参数，提取点级特征。

新手可以先把它理解成：

```text
对每个点用同一个小网络提特征
再把所有点的特征汇总
```

### 3.4 PointNet 的局限

PointNet 很简洁，但它对局部结构建模不够强。

例如：

```text
椅子腿附近的一小片点
桌面边缘附近的一小片点
车道线的一段局部几何
```

这些局部结构对任务很重要，而 PointNet 原始形式更偏向直接得到全局特征。

## 4. PointNet++

### 4.1 PointNet++ 改进了什么

PointNet++ 的核心改进是：

```text
PointNet++:
Improvement of PointNet
Multiscale grouping (local grouping points)
```

也就是说，PointNet++ 的核心改进是引入局部区域建模。

### 4.2 Local grouping 是什么

点云中某个点附近的一圈点可以组成一个局部邻域。

```text
中心点 -> 找附近点 -> 形成局部 group -> 提取局部特征
```

这样模型不仅能看全局，也能看局部结构。

### 4.3 Multiscale grouping 是什么

不同尺度表示不同大小的邻域。

```text
小尺度：看很近的点，关注细节
大尺度：看更大范围，关注整体结构
```

多尺度分组可以让模型同时学习：

```text
局部细节 + 较大范围结构
```

这对点云很重要，因为点云密度经常不均匀。

## 5. PointNet Application：自动驾驶预测和规划

这里的应用方向是：

```text
Autonomous Driving Prediction and ML Planning
PointNet for subgraph feature extraction
```

### 5.1 自动驾驶里为什么需要点云或向量特征

自动驾驶场景里，不只有相机图像，还会有：

```text
车道线
路口拓扑
行人轨迹
车辆轨迹
地图元素
LiDAR 点云
```

这些数据很多不是规则图片，而更像点、线、图结构。

### 5.2 subgraph feature extraction 是什么

可以把道路或地图拆成很多局部子图：

```text
一段车道线
一个路口区域
一组相邻车辆轨迹
```

PointNet 可以用来给这些局部结构提取特征。

```text
局部点/线/轨迹 -> PointNet -> 子图特征
```

这些特征再送入后续模块，用于：

```text
轨迹预测
交互建模
规划决策
地图补全
```

## 6. 把整节课串成一个流程

如果只记一张图，可以记这个：

```text
2D 图像：
图片 -> Conv2d/CNN/ViT -> 特征 -> 分类/检测/分割/位姿估计

训练：
数据 -> 模型 -> loss -> backward -> optimizer.step()

推理：
读取图像 -> 预处理 -> 加载模型 -> forward -> 输出结果

部署：
训练模型 -> ONNX/TensorRT/OpenVINO/TFLite -> 量化/加速 -> 目标设备运行

3D 点云：
N * 3 点 -> PointNet/PointNet++ -> 全局或局部特征 -> 分类/分割/预测/规划
```
