# L2-3 操作与抓取算法：新手学习课程

---

## 0. 这节课在讲什么

本节主要讲两件事：

1. **机器人抓取与操作的基础概念**
   - manipulation 是什么
   - grasping 是什么
   - 接触点、摩擦锥、wrench、force closure、form closure 分别是什么意思

2. **机器人操作任务的工程案例**
   - pick-and-place：抓取并放置
   - 理疗按摩：视觉定位 + 轨迹规划 + 力控
   - insertion/thread removal：插入、接触检测、末端工具执行

用一句话概括：

```text
机器人操作 = 机器人通过接触改变物体或环境的状态
机器人抓取 = 机器人通过接触点施加力和力矩，让物体被稳定约束
```

新手学习这节课时，不要一开始就纠结所有数学细节。先抓住三个问题：

```text
1. 机器人和物体在哪里接触？
2. 每个接触点能施加什么样的力？
3. 这些力合起来能不能抵抗外界扰动？
```

## 1. Manipulation 和 Grasping

### 1.1 Manipulation：机器人操作

Manipulation 指机器人通过自己的运动和接触，对物体或环境产生影响。

例子：

| 任务 | 机器人做了什么 |
|---|---|
| 把杯子拿起来 | 抓住杯子并移动 |
| 推动物体到指定位置 | 用末端或手指推物体 |
| 插入零件 | 对准孔位并沿轨迹插入 |
| 拧螺丝、打磨、按摩 | 持续接触并控制力 |

所以 manipulation 不只是“拿东西”，也包括推动、滑动、插入、接触式加工等任务。

### 1.2 Grasping：机器人抓取

Grasping 是 manipulation 的一个子集。

Grasping 的定义可以理解为：

```text
Grasping = 在若干接触点施加力和力矩，从而限制物体运动
```

更直白地说：

```text
不是夹爪碰到物体就叫抓住；
只有物体在受到干扰时仍不容易掉、不容易滑、不容易转，才算抓得稳定。
```

目前很多抓取研究都从相对简单的静态场景开始：

```text
物体静止
环境基本不变
相机看到物体
机器人计算抓取姿态
夹爪执行抓取
```

## 2. 抓取问题为什么难

抓取问题有两个主要挑战。

### 2.1 接触建模很难

机器人抓物体时，真实接触非常复杂：

- 接触面可能不是一个点，而是一小片区域
- 物体表面可能粗糙、光滑、软、硬
- 摩擦力大小不一定准确
- 物体可能发生微小滑动
- 夹爪、传感器、标定都有误差

为了让问题能算，算法通常会做假设。例如：

```text
把接触简化成点接触
把摩擦简化成摩擦锥
假设物体刚性
假设接触点位置准确
```

这些假设让数学模型变简单，但也会带来工程风险。

### 2.2 搜索空间维度很高

抓取算法要回答：

```text
在哪里抓？
用什么角度抓？
夹爪张多大？
从哪个方向接近？
抓取后怎么移动？
```

如果是 2D 平面抓取，常见表示是：

```text
x, y, w, h, theta
```

含义：

| 符号 | 含义 |
|---|---|
| x, y | 抓取中心位置 |
| w | 夹爪宽度 |
| h | 抓取框高度或有效区域 |
| theta | 抓取角度 |

如果是 6DoF 抓取，抓取位姿更复杂：

```text
x, y, z, roll, pitch, yaw
```

这里的 6 个量合起来描述一个完整的三维抓取姿态：

| 符号 | 含义 | 新手理解 |
|---|---|---|
| x | 左右方向的位置 | 夹爪抓取中心在空间中横向在哪里 |
| y | 前后方向的位置 | 夹爪离机器人远近在哪里 |
| z | 上下方向的位置 | 夹爪要到多高的位置 |
| roll | 绕 x 轴旋转 | 夹爪像“歪头”一样左右倾斜 |
| pitch | 绕 y 轴旋转 | 夹爪像“点头/抬头低头”一样调整俯仰 |
| yaw | 绕 z 轴旋转 | 夹爪像“左右转头”一样改变朝向 |

简单记：

```text
x, y, z = 抓取位置
roll, pitch, yaw = 抓取朝向
```

因此抓取本质上是一个高维搜索和优化问题。

## 3. 接触模型：机器人到底能施加什么力

### 3.1 Point Contact：点接触

点接触把夹爪和物体之间的接触简化成一个点。

#### 无摩擦点接触

Frictionless point contact 只考虑法向力。

直觉：

```text
手指只能“顶住”物体，不能靠摩擦“拽住”物体。
```

这种模型很保守，因为现实中很多抓取都依赖摩擦。

#### 有摩擦点接触

Friction point contact 同时考虑：

- 法向力
- 摩擦力

这时就会出现一个重要概念：**friction cone，摩擦锥**。

### 3.2 Friction Cone：摩擦锥

摩擦锥表示在不打滑的前提下，一个接触点能施加的所有可能力的方向范围。

可以这样理解：

```text
如果力的方向落在摩擦锥里面，接触点不容易滑。
如果力的方向超出摩擦锥，物体就可能打滑。
```

摩擦越大，摩擦锥越“宽”；摩擦越小，摩擦锥越“窄”。

例子：

| 场景 | 摩擦锥直觉 |
|---|---|
| 橡胶夹爪夹木块 | 摩擦大，能承受更斜的力 |
| 金属夹爪夹玻璃 | 摩擦小，更容易滑 |
| 湿滑表面 | 摩擦锥变窄，抓取风险变大 |

### 3.3 Soft Contact / Patch Contact：软接触或面接触

真实夹爪和物体之间常常不是一个点，而是一小片接触区域。

这种模型更接近现实，但计算也更复杂。

patch contact 的一个评价思路是：

```text
根据点云中每个点的法向方向与夹爪方向是否对齐来打分。
```

新手可以先理解为：

```text
夹爪越能贴合物体表面，抓取通常越稳定。
```

## 4. Wrench：力和力矩的合体

### 4.1 什么是 Wrench

Wrench 可以简单理解为：

```text
wrench = force + torque
```

也就是：

```text
力 + 力矩
```

为什么抓取不能只看力？

因为物体不仅会平移，还会旋转。

例子：

```text
你用手指从杯子侧面推一下，杯子可能不只是往旁边移动，还可能发生转动。
```

所以评价抓取时，要同时考虑：

- 能不能抵抗外力导致的平移
- 能不能抵抗外力矩导致的旋转

### 4.2 接触点到质心的向量

第 i 个 contact point 的 wrench，和接触点到 COM 的向量有关。

COM 是 center of mass，也就是质心。

直觉：

```text
同样大小的力，作用位置离质心越远，越容易产生旋转。
```

这和开门类似：

```text
推门把手比推门轴附近更容易把门推开。
```

### 4.3 Grasp Wrench Space

Grasp Wrench Space 指：

```text
所有接触点在允许力范围内，能对物体产生的所有 wrench 的集合。
```

更直白：

```text
这个抓取姿态到底能“对抗”哪些外界干扰。
```

如果这个空间很大，说明抓取得比较稳；如果这个空间很小，说明一有扰动就可能失败。

## 5. 如何评价一个抓取好不好

抓取评价里有两个核心概念：

```text
Form Closure
Force Closure
```

这两个词很重要。

### 5.1 Form Closure：形状约束

Form closure 的意思是：

```text
只靠几何约束，就让物体无法运动。
```

它通常不依赖摩擦力。

直觉例子：

```text
物体被多个刚性挡块从不同方向卡住，即使没有摩擦，也动不了。
```

这里要注意：

| 场景 | 通常需要的接触点数量 |
|---|---|
| 2D | 4 个接触点 |
| 6D | 7 个接触点 |

新手要记住：

```text
Form Closure 更像“用形状把物体锁死”。
```

### 5.2 Force Closure：力约束

Force closure 的意思是：

```text
对于任意外部 wrench，都存在一组接触力可以把它抵消。
```

直白地说：

```text
不管外界从哪个方向推、拉、扭，夹爪都能通过接触力把它平衡住。
```

Force closure 会考虑摩擦。

所以它比 form closure 更贴近很多真实夹爪抓取问题。

### 5.3 Form Closure 和 Force Closure 的区别

| 对比项 | Form Closure | Force Closure |
|---|---|---|
| 主要依赖 | 几何约束 | 接触力和摩擦 |
| 是否考虑摩擦 | 通常不考虑 | 考虑 |
| 直觉 | 卡住物体 | 用力和摩擦稳住物体 |
| 严格程度 | 通常更严格 | 通常更灵活 |
| 接触点需求 | 可能更多 | 可能更少 |

一句话记忆：

```text
Form Closure 看“有没有被几何锁死”；
Force Closure 看“能不能用力抵抗所有扰动”。
```

## 6. Force Closure 的计算直觉

基于 force closure 的抓取计算流程可以这样理解。

### 6.1 关键对象

| 名称 | 新手理解 |
|---|---|
| Force cone | 单个接触点允许施加的力范围 |
| Wrench space | 接触点产生的力和力矩范围 |
| Grasp Wrench Hull | 多个 wrench 组合形成的凸包 |

### 6.2 基本计算流程

以 2D 场景为例，可以这样理解：

```text
1. 找到每个接触点
2. 根据摩擦系数计算每个接触点的 force cone
3. 把 force cone 转换成 wrench
4. 把所有接触点的 wrench 组合起来
5. 看原点是否在 Grasp Wrench Hull 内部
```

为什么要看原点？

因为原点代表：

```text
合力和合力矩为 0
```

如果 wrench hull 包含原点，意味着机器人有可能组合出相互平衡的接触力，并进一步抵抗外部扰动。

### 6.3 Force Closure 质量指标

一个常见质量指标是：

```text
以原点为中心，能放进 Grasp Wrench Hull 的最大球的半径
```

新手可以理解成：

```text
这个球越大，说明抓取能抵抗的外界扰动越大。
```

如果球很小：

```text
稍微有一点外力，抓取就可能到达极限。
```

## 7. Form Closure 的计算直觉

Form closure 关注的是物体是否被静止约束完全限制住。

Form closure 的评估过程大致是：

```text
1. 构建每个接触点的约束关系
2. 建立线性规划问题
3. 通过 linear programming 计算参数 k
4. 判断是否满足 form closure
```

这里不要求新手马上掌握线性规划细节。先理解它的本质：

```text
把每个接触点能阻止物体运动的方向写成数学约束，
再检查这些约束合起来能不能阻止所有可能运动。
```

学习建议：

```text
先理解“接触点限制运动”的几何直觉，
再去补线性代数和线性规划。
```

## 8. 常见抓取任务类型

### 8.1 多指抓取

Multi-finger grasping 使用多个手指接触物体。

优点：

- 灵活
- 能适应复杂形状
- 有机会实现更稳定的抓取

难点：

- 控制复杂
- 接触点多，建模和优化更难
- 硬件成本和标定难度更高

### 8.2 两指夹爪抓取

Two-finger gripper 是工业和实验室中很常见的方案。

优点：

- 结构简单
- 控制相对容易
- 工程可靠性高

常见表示：

```text
Planar grasping: x, y, w, h, theta
6DoF grasping: x, y, z, roll, pitch, yaw
```

其中 6DoF grasping 里的 `x, y, z` 表示夹爪在三维空间中的位置，`roll, pitch, yaw` 表示夹爪的三维朝向。

新手可以先从 2D 平面抓取入门，再理解 6DoF 抓取。

## 9. Planar Pushing：平面推动

robotic manipulation 中还有一个重要任务：planar pushing。

### 9.1 什么是平面推动

平面推动就是机器人在平面上推物体，让物体移动或转动。

例如：

```text
机器人用末端执行器把桌面上的方块推到目标位置。
```

### 9.2 Quasi-static：准静态

quasi-static 指的是：

```text
Force balance + slow motion
```

新手理解：

```text
物体动得很慢，可以近似认为每一瞬间都接近平衡状态。
```

这样可以简化动力学问题。

### 9.3 为什么推动也重要

有些物体不好直接抓：

- 太大
- 太扁
- 太重
- 姿态不适合夹爪进入

这时机器人可以先推一下，把物体调整到更容易抓的位置。

所以真实操作任务往往不是单纯 grasping，而是：

```text
pushing + grasping + placing + insertion + force control
```

## 10. 案例 1：Pick-and-place

### 10.1 问题

机器人要把物体从一个位置抓起来，再放到另一个位置。

这个任务的输入假设：

```text
RGBD 数据
已知物体类别
需要做 3D 位姿估计
```

RGBD 表示：

```text
RGB 图像 + Depth 深度图
```

### 10.2 典型流程

```text
1. 视觉感知：识别物体并估计位姿
2. 计算 2D 平面抓取
3. 用简单规则生成抓取姿态
4. 规划笛卡尔轨迹
5. 转换成关节轨迹
6. 执行抓取
7. 移动到目标位置
8. 放置物体
```

### 10.3 工程讨论

这种方案通常依赖较多假设：

- 物体类别已知
- 场景比较干净
- 目标物体能被稳定识别
- 抓取姿态生成规则适用于当前物体
- 运动过程中不需要复杂避障

优点：

```text
在符合假设的场景中比较稳定。
```

缺点：

```text
泛化性一般，场景稍微变化就可能需要重新调规则。
```

## 11. 案例 2：理疗按摩

### 11.1 问题

机器人执行理疗按摩，本质是一个接触式操作任务。

它不仅要到达位置，还要控制接触力。

### 11.2 流程

```text
1. 视觉识别目标区域
2. 从 2D 图像投影到 3D 坐标
3. 生成按摩轨迹
4. 规划并执行轨迹
5. 叠加力控
```

### 11.3 工程难点

因为机器人要接触人体，所以要求很高：

- 感知识别要稳定
- 轨迹要平滑
- 力控要可靠
- 安全边界必须明确
- 失效处理必须保守

这类任务说明：

```text
机器人操作不只是“算一个位置”，还要考虑接触力、安全性和工程稳定性。
```

---

## 12. 案例 3：Insertion & Thread Removal

### 12.1 问题

任务是移除航空元器件螺孔内部的螺纹。

这类任务通常要求：

- 高精度定位
- 稳定接触检测
- 工具路径准确
- 执行过程可靠

### 12.2 流程

```text
1. 标定：手眼、相机内参、工具参数
2. 视觉定位：根据孔径推算 3D 位置
3. 路径规划：机器人移动到孔附近
4. 力传感器判断进入和接触
5. 启动末端工具
6. 完成操作后移出
```

### 12.3 工程讨论

这种方法偏传统：

- 规则多
- 先验多
- 泛化能力弱
- 对精度和稳定性要求高

但在受控工业场景中，传统方法经常很有价值。

原因是：

```text
工业落地首先看稳定性和可验证性，不只看算法是否新。
```

## 13. 产品化视角：为什么实验室 Demo 不等于产品

机器人操作系统还需要考虑场景、任务、精度、可靠性和商业化。

### 13.1 场景越受控，越容易落地

| 场景类型 | 落地难度 |
|---|---|
| 固定工位、固定物体、固定流程 | 较低 |
| 有限物体类别、有限变化 | 中等 |
| 开放家庭环境、物体随意摆放 | 很高 |

场景越开放，感知、规划、控制、测试都会变难。

### 13.2 精度和可靠性是非线性的

机器人系统常常存在“非线性”的精度要求。

可以这样理解：

```text
从 80% 成功率做到 90% 可能不难；
从 95% 做到 99.5% 可能非常难；
从 99.5% 做到可商业化交付，难度又会再上一个台阶。
```

机器人产品尤其如此，因为失败可能带来：

- 物体掉落
- 工具损坏
- 设备停线
- 人员安全风险

### 13.3 从研发到商业化

从研发到商业化通常可以理解为：

```text
预研 -> Demo -> 小批量交付 -> 规模化交付 -> 商业化
```

每个阶段关注点不同：

| 阶段 | 重点 |
|---|---|
| 预研 | 证明技术方向可行 |
| Demo | 展示核心能力 |
| 小批量交付 | 找真实问题、修工程细节 |
| 规模化交付 | 稳定性、成本、维护 |
| 商业化 | 客户价值、供应链、服务体系 |
