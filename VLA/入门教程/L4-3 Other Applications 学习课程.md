# L4-3 Other Applications：新手学习课程

---

## 0. 这节课在讲什么

前面已经学习了 2D 平面抓取和 6DoF 抓取。本节继续看抓取学习里的其他方向。

这些方向不再只关注“给一个物体生成一个抓取姿态”，而是进一步考虑：

- 能不能直接从视觉输入学机器人动作
- 能不能在杂乱环境里连续抓取
- 能不能根据语言指令抓指定物体
- 能不能做移动抓取、全身抓取、动态抓取
- 能不能用灵巧手和多指手完成更复杂的抓取

可以把本节分成两大部分：

```text
其他抓取应用
Dexterous / Multi-finger grasping
```

---

## 1. Visuomotor Policy for Grasping

Visuomotor policy 指的是从视觉输入直接输出机器人动作的策略。

传统抓取系统通常会分成很多模块：

```text
视觉感知 -> 3D 投影 -> 位姿估计 -> 抓取姿态生成 -> 路径规划 -> 轨迹生成 -> 执行
```

visuomotor policy 的思路更直接：

```text
图像输入 -> 神经网络策略 -> 机器人动作
```

它的特点是：

- 不一定需要显式做 3D 投影
- 不一定需要单独做运动规划
- 不一定需要手写轨迹生成
- 可以通过大量数据学习视觉和动作之间的关系

这个方向也可以叫 hand-eye coordination，也就是手眼协调学习。

### 1.1 它的优点

优点是系统链路短，端到端程度高。如果数据足够多，模型可能学到传统模块化系统里很难手写的策略。

例如，机器人看到当前图像后，直接预测下一步夹爪应该往哪里移动、是否闭合夹爪。

### 1.2 它的问题

visuomotor policy 不一定是抓取问题的最佳范式。

原因是：

- 需要大量真实或仿真数据
- 可解释性较弱
- 泛化到新场景不一定稳定
- 安全性和失败恢复更难验证
- 很多工程场景仍然需要规划、约束和安全边界

所以它更像是一条重要研究路线，而不是所有抓取场景的默认答案。

## 2. Visual Servoing

Visual servoing 可以理解成基于视觉反馈的机器人运动控制。

它不是一次性预测完整抓取动作，而是在执行过程中不断看图像，根据视觉误差调整机器人运动。

直观流程是：

```text
观察当前图像
-> 比较目标状态和当前状态
-> 根据视觉误差调整机器人动作
-> 再观察、再调整
```

例如夹爪接近物体时，如果发现目标在图像里偏左，就控制末端向左修正；如果目标变大，说明正在接近。

visual servoing 的优点是可以利用闭环反馈，减少一次性估计误差的影响。缺点是对视觉稳定性、控制频率和时延比较敏感。

## 3. Bin-picking 和杂乱环境抓取

bin-picking 指从箱子或料筐里抓取物体。

这个问题比单个物体抓取难很多，因为环境通常是：

- 多个物体堆叠
- 物体互相遮挡
- 抓一个物体会影响其他物体
- 当前最好抓的物体会随着动作变化

因此 bin-picking 往往不是单步问题，而是序列决策问题。

可以理解成：

```text
当前状态 -> 选择一个动作 -> 环境变化 -> 再选择下一个动作
```

动作不一定只有抓取，也可以包括推动。

例如：

```text
先推开挡住目标的物体
再抓取暴露出来的物体
```

这类问题常用强化学习、自监督学习或序列策略学习来建模。

---

## 4. Push and Grasp

push and grasp 是杂乱环境中很常见的组合策略。

单纯抓取有时会失败，因为目标物体被挡住、夹爪插不进去，或者物体姿态不适合抓。

这时机器人可以先推一下，让场景变得更容易抓。

典型逻辑是：

```text
当前图像
-> 判断直接抓取是否合适
-> 如果不合适，先选择推动动作
-> 场景变化后再尝试抓取
```

这说明机器人操作不一定是“只抓”。在真实场景中，推动、拨开、分离、整理经常是抓取成功的前置动作。

## 5. Language Guided Grasping

Language guided grasping 指根据语言指令进行抓取。

例如用户说：

```text
抓红色杯子
抓左边那个盒子
抓可以倒水的容器
```

机器人需要把语言描述和视觉场景对齐，找到目标物体，再生成抓取动作。

这种任务通常包含两个关键部分：

1. 语言理解：理解目标描述
2. 视觉定位和抓取：在图像或点云里找到对应目标并抓取

它比完整的 VLA 任务简单一些。VLA 通常指 vision-language-action，需要模型从视觉和语言直接规划动作序列；language guided grasping 更聚焦于“根据语言找到并抓取目标”。

### 5.1 为什么语言有用

语言可以给机器人提供额外信息。

例如一个桌面上有多个杯子，仅靠视觉可能不知道用户想要哪一个；语言描述可以指定颜色、位置、类别或功能。

### 5.2 难点

难点包括：

- 语言描述可能模糊
- 多个物体可能满足描述
- 目标可能被遮挡
- 模型需要融合视觉和语言特征
- 抓取还要满足几何和碰撞约束

## 6. 其他抓取方向

### 6.1 Whole-body grasping

Whole-body grasping 指不只用手或夹爪，而是利用身体更多部位完成抓取或支撑。

例如人抱大箱子时，不只是手指在工作，手臂、胸部甚至身体姿态都参与了稳定物体。

机器人做类似任务时，需要考虑全身姿态、接触点和稳定性。

### 6.2 Mobile grasping

Mobile grasping 指移动机器人上的抓取。

它不仅要控制机械臂，还要控制移动底盘。

问题会变成：

```text
机器人先移动到哪里？
底盘和机械臂怎么配合？
怎么在不碰撞的情况下接近目标？
```

移动抓取比固定机械臂抓取更复杂，因为机器人本体位置也会影响可达性和抓取稳定性。

### 6.3 Dynamic grasping

Dynamic grasping 指抓取运动中的物体。

例如物体在传送带上移动，机器人需要预测目标未来位置，并在合适时机抓取。

这类任务需要考虑：

- 目标运动预测
- 抓取时机
- 快速运动规划
- 控制延迟
- 失败恢复

## 7. Dexterous & Multi-finger Grasping

Dexterous grasping 指灵巧手抓取，multi-finger grasping 指多指抓取。

相比两指夹爪，灵巧手可以做更复杂的操作。例如：

- 用多个手指包住物体
- 调整物体在手中的姿态
- 模仿人手抓取
- 完成更精细的接触任务

但它也更难。

两指夹爪通常只需要考虑夹爪位置、朝向和开合宽度；灵巧手还要考虑每个手指关节角度。

灵巧手抓取姿态常写成：

```text
G = [R, t, q]
```

其中：

| 符号 | 含义 |
|---|---|
| R | 手或掌心的旋转 |
| t | 手或掌心的位置 |
| q | 手指关节角 |

如果是简化表示，也可能写成：

```text
G = [R, t, w, c]
```

其中 `w` 可以表示抓取宽度，`c` 可以表示抓取类型。

## 8. 灵巧手抓取的主要任务

灵巧手抓取不只是预测一个位姿，它通常涉及多个子问题。

### 8.1 Grasp pose generation

生成手的整体位姿和手指关节角。

也就是要回答：

```text
手掌应该放在哪里？
手应该朝哪个方向？
每个手指应该弯多少？
```

### 8.2 Contact modeling

灵巧手和物体之间有多个接触点。

每个手指的接触位置、接触力和摩擦都会影响抓取稳定性。

接触建模比两指夹爪复杂得多。

### 8.3 Human hand to dexterous hand retargeting

很多数据来自人手抓取，但机器人灵巧手和人手结构不同。

retargeting 的意思是：

```text
把人手动作转换成机器人手可以执行的动作
```

这需要考虑手指数量、关节结构、运动范围和接触一致性。

### 8.4 Grasping control policy

除了静态抓取姿态，灵巧手还需要控制策略。

例如从张开手到闭合手的过程，每个关节如何运动、如何根据接触反馈调整，都属于 policy 问题。

## 9. GSNet 表示思路

GSNet 一类方法关注如何高效表示多指抓取。

核心思路是：先为抓取姿态和抓取类型建立表示，再用决策模型给候选抓取打分。

简化后可以理解成：

```text
输入物体信息
-> 生成候选多指抓取
-> 对抓取姿态和类型评分
-> 选择较好的抓取
```

这种方法强调 representation，也就是如何把复杂的多指抓取表达成模型容易学习和评估的形式。

---

## 10. UniDexGrasp

UniDexGrasp 面向通用灵巧手抓取。

它通常包含两部分：

1. Grasp proposal generation
2. Grasping policy

Grasp proposal generation 负责生成候选抓取，包括：

- rotation
- translation
- joint angle
- optimization

Grasping policy 负责学习如何执行抓取，可能用到：

- reinforcement learning
- distillation
- DAgger

这里的抓取表示仍然可以写成：

```text
g = (R, t, q)
```

其中 `q` 是灵巧手的关节角。

---

## 11. CGF：Continuous Grasping Function

CGF 关注从人类示范中学习连续的灵巧手抓取函数。

它的形式可以写成：

```text
q(t) = f(t, z, Fo; theta)
```

可以这样理解：

- `q(t)` 是某个时间点的手指关节状态
- `t` 表示时间
- `z` 是潜变量或动作表示
- `Fo` 表示物体相关特征
- `theta` 是模型参数

这种方法不是只输出一个静态手势，而是学习随时间变化的抓取过程。

模型上可能会用到：

- retargeting
- PointNet
- MLP
- continuous grasping function

它适合处理从人手演示到机器人灵巧手动作的转换。

---

## 12. GraspTTA

GraspTTA 关注手和物体接触的一致性，并使用 test-time adaptation。

它包含的组件包括：

- GraspVAE
- ContactNet
- TTA

其中 TTA 是 test-time adaptation，也就是测试时适应。

直觉是：模型在测试时根据当前物体和接触情况做调整，让生成的手势更符合接触约束。

这类方法会显式考虑 contact model，例如手指和物体表面是否真的接触、接触区域是否合理。

## 13. 灵巧手数据集和评价

灵巧手抓取需要大量数据。

常见数据包括：

| 数据集 | 特点 |
|---|---|
| DexYCB | 真实世界数据，包含大量人手抓取 YCB 物体 |
| DexGrabNet | 仿真数据，使用 ShadowHand，包含大量物体和抓取 |

评价指标中，最直接的是 success rate。

但灵巧手任务还经常关注：

- 是否稳定抓住
- 接触是否合理
- 手指是否穿模
- 轨迹是否平滑
- 是否能泛化到新物体

## 14. 三类抓取学习怎么对比

可以把 2D 抓取、6DoF 抓取、灵巧手抓取放在一起看。

| 类型 | 抓取表示 | 常见输入 | 难点 |
|---|---|---|---|
| 2D Planar Grasping | `(x, y, w, h, theta)` | top-view RGB-D | 主要在像素坐标里学习 |
| 6DoF Grasping | `(R, t, w)` | 点云、RGB-D | 需要三维几何和姿态 |
| Dex / Multi-finger | `(R, t, q)` | 点云、RGB、手模型 | 手指关节、接触、retargeting、控制策略 |

难度大致可以这样理解：

```text
2D 抓取：决定图像上哪里抓
6DoF 抓取：决定三维空间中怎么抓
灵巧手抓取：决定手掌和每个手指怎么配合抓
```

越往后，表示空间越大，接触越复杂，控制也越难。

## 15. 和前面课程的关系

这节内容把抓取学习从“单次抓取姿态预测”扩展到了更真实的机器人操作场景。

可以这样串起来：

```text
2D grasping：从图像上预测抓取框
6DoF grasping：从点云或 RGB-D 中预测三维抓取位姿
bin-picking：在杂乱环境中连续选择动作
language guided grasping：根据语言指定目标
visuomotor policy：从视觉直接学动作策略
dexterous grasping：用多指手建模复杂接触和控制
```

真实机器人系统往往不是单独使用某一种方法，而是把感知、语言、抓取候选生成、抓取评价、运动规划和控制策略组合起来。
