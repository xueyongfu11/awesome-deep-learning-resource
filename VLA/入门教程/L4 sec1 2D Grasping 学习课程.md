# L4 sec1 2D Grasping：新手学习课程

---

## 0. 这节课在讲什么

本节围绕 **2D 平面抓取** 展开。

机器人抓取可以有很多形式：两指夹爪抓取、多指灵巧手抓取、6DoF 抓取等。2D 平面抓取是其中比较适合入门的一类，因为它把抓取问题简化到了图像平面上。

核心问题可以理解为：

```text
给一张 RGB 或 RGB-D 图像，
让模型预测夹爪应该从图像中的哪个位置、以什么角度、张多宽去抓。
```

常见输出是一个抓取矩形：

```text
x, y, w, h, theta
```

含义如下：

| 符号 | 含义 |
|---|---|
| x, y | 抓取中心在图像中的位置 |
| w | 夹爪张开的宽度 |
| h | 抓取矩形的高度 |
| theta | 抓取角度 |

新手可以先把 2D grasping 理解成一种“图像上的抓取框预测”问题。

---

## 1. 2D 平面抓取在机器人系统里的位置

2D 平面抓取通常不是完整机器人系统的终点。它只是给出一个图像平面上的抓取建议。

真实机器人执行时，还需要把图像中的抓取结果变成机器人坐标系里的动作。大致流程是：

```text
RGB / RGB-D 图像
-> 预测 2D 抓取矩形
-> 根据深度信息投影到 3D 空间
-> 转换到机器人基坐标系
-> 生成预抓取位姿
-> 路径规划和轨迹执行
```

这里有一个关键点：如果要从 2D 图像位置变成 3D 空间位置，通常需要深度信息。

只看 RGB 图像，模型可以预测“从图像哪里抓”，但机器人还不知道这个点离相机多远。RGB-D 图像或深度相机能提供距离信息，因此在机器人抓取里很常见。

---

## 2. Grasping Problem 的基本分类

抓取问题可以按夹爪类型和输出形式来理解。

常见术语包括：

| 术语 | 含义 |
|---|---|
| Antipodal grasping | 两个接触点大致相对，适合两指夹爪 |
| Parallel gripper | 平行两指夹爪 |
| Dexterous grasping | 灵巧手抓取 |
| Multi-finger grasping | 多指抓取 |

如果关注两指夹爪，常见问题包括：

- 2D planar grasping：在图像平面预测抓取矩形
- 6DoF grasping：在三维空间预测完整抓取位姿

如果关注灵巧手或多指手，问题会更复杂，因为要考虑多个手指、多个接触点和更高维控制。

---

## 3. 学习方法的输入和输出

2D grasping 的学习输入通常是：

- RGB 图像
- 深度图
- RGB-D 图像

有些更复杂的方法也可能使用点云、mesh 或其他传感器信息。

学习输出通常有两类：

```text
2D grasping pose
6DoF grasping pose
```

在本节中，重点是 2D 抓取姿态，也就是抓取矩形。

另外，有些方法不直接输出一个抓取框，而是输出：

- 每个像素的抓取质量
- 抓取角度图
- 抓取宽度图
- 分割 mask
- affordance map

这些输出最后都可以转成机器人可用的抓取候选。

---

## 4. 2D Planar Grasping 的核心表示

2D 平面抓取最常见的表示是矩形：

```text
(x, y, w, h, theta)
```

它可以想象成一只两指夹爪从图像上某个位置夹过去。

其中：

- `x, y` 决定夹爪中心
- `theta` 决定夹爪旋转角度
- `w` 决定两指张开距离
- `h` 表示抓取区域的高度或夹爪接触区域

如果模型输出的是每个像素的质量、角度和宽度图，那么可以选质量最高的像素，再读取该像素对应的角度和宽度，生成最终抓取矩形。

---

## 5. GG-CNN

GG-CNN 是一个比较直观的 2D 抓取模型。

它的输入是深度图，模型结构主要由简单的 `Conv2d` 层组成。

输出不是一个单独的抓取框，而是多个 dense map。可以理解成模型对每个像素都预测抓取信息。

常见输出包括：

- position / quality：这个位置适不适合抓
- angle：夹爪角度
- width：夹爪宽度

因此 GG-CNN 的推理逻辑是：

```text
输入深度图
-> 网络输出每个像素的抓取质量、角度、宽度
-> 找到质量最高的位置
-> 生成对应抓取矩形
```

训练时常用 MSE loss，让预测图和标注图之间的误差变小。

新手可以把 GG-CNN 理解成：

```text
不是只预测一个框，而是在整张图上预测哪里适合抓。
```

---

## 6. GR-ConvNet

GR-ConvNet 和 GG-CNN 思路相近，也属于直接回归抓取结果的方法。

它的全称是 Generative Residual Convolutional Neural Network，用于 antipodal robotic grasping。

它可以使用 RGB-D 输入，也可以只使用 RGB。模型使用卷积网络提取特征，并加入 residual 结构，让网络更容易训练。

输出 head 通常包括：

- quality
- angle
- width

其中角度常用两个 head 表示，避免角度周期性带来的训练问题。

GR-ConvNet 的 loss 常用 Smooth L1 loss。相比普通 MSE，Smooth L1 对异常误差更稳一些。

### 6.1 GR-ConvNet 怎么评价

常见评价方式是看预测抓取矩形和真实抓取矩形是否足够接近。

一个预测通常被认为成功，需要满足：

```text
预测框和真实框 IoU > 25%
预测角度和真实角度差 < 30 度
```

这套评价方式在 Cornell 等 2D 抓取数据集上很常见。

---

## 7. NBMOD

NBMOD 的思路更接近检测任务。

它输入一张固定大小的图像，例如 `416 x 416`，输出抓取相关的置信度和矩形参数：

```text
confidence, x, y, w, h, theta
```

这和目标检测有点像：目标检测预测物体框，NBMOD 预测抓取框。

区别是，抓取框不只是物体在哪里，还要表达夹爪应该怎么夹。

训练时会区分 positive samples 和 negative samples，并用 MSE 做回归。

新手可以这样理解：

```text
NBMOD 把抓取检测做得很像一个目标检测问题。
```

---

## 8. Grasp-Det-Seg

Grasp-Det-Seg 是一种多任务方法。

它不只做抓取检测，还会结合 segmentation 和 refinement。

核心模块包括：

- RPN：生成候选区域
- grasp detection：预测抓取
- segmentation：分割目标
- refinement：进一步修正结果

这种方法的思路是：抓取检测不是孤立任务。如果模型同时知道物体区域在哪里、语义分割结果是什么，就可能更好地预测抓取。

loss 也会由多个部分组成：

- grasp detection loss
- segmentation loss
- refinement loss

---

## 9. DexNet 2.0

DexNet 2.0 是 Berkeley 的经典工作之一。

它的重点不是直接预测一个抓取框，而是学习 **grasp quality**，也就是判断一个候选抓取好不好。

它的流程可以理解为：

```text
先用传统方法采样候选抓取
再用网络评估每个候选抓取的质量
选择质量高的抓取执行
```

DexNet 2.0 的训练数据规模很大，包含数百万级数据点。标签来自 analytic grasp evaluation，也就是通过解析指标自动评估抓取质量。

这个思路很重要，因为它把抓取分成了两件事：

```text
生成候选抓取
评价候选抓取
```

很多抓取方法都可以从这个角度理解。

---

## 10. 抓取区域、分割和 Affordance

有些方法不直接回归一个抓取矩形，而是先学习哪里可以抓。

例如：

- 预测 grasp affordance segmentation
- 找到可能抓取区域
- 再从区域里生成抓取矩形

Affordance 可以理解成“可操作性”。在抓取里，它表示图像中哪些区域适合被夹爪抓。

这种思路更接近：

```text
先判断哪里能抓
再生成具体怎么抓
```

它也可以拆成 segmentation 和 classification 等子任务。

---

## 11. 常见 2D Grasping 数据集

### 11.1 Cornell

Cornell 是经典 2D 抓取数据集。

它包含 1035 张 RGB-D 图像，分辨率为 `640 x 480`，覆盖 240 个真实物体。

标注包括：

- 5110 个 positive grasps
- 2909 个 negative grasps

Cornell 数据集规模不大，但在早期 2D 抓取研究里非常常见。

### 11.2 Jacquard

Jacquard 是更大规模的数据集。

它包含约 54k 张 RGB-D 图像，抓取标注来自仿真环境中的抓取尝试。

总抓取样本约 1.1M。

Jacquard 的优点是规模大，适合训练深度模型；缺点是仿真数据和真实环境之间可能存在差距。

### 11.3 Grasp Anything

Grasp Anything 是更偏合成和扩展数据的方向。

它使用语言 prompt 生成或组织数据，并借助预训练模型和后处理进行标注优化。

这个方向说明一件事：

```text
当模型规模变大时，数据生成和标注方式会变得非常关键。
```

## 12. 2D Grasping 的评价指标

2D 抓取评价可以有三种方式：

- 真实机器人实验
- 仿真环境
- 解析抓取评价指标

数据集上常见的成功标准是：

```text
预测抓取框和正样本标注框 IoU > 25%
预测角度和标注角度差 < 30 度
```

真实实验中，评价更直接：

```text
success rate = 成功抓取次数 / 总尝试次数
```

需要注意的是，数据集指标和真实机器人成功率不完全等价。

模型在数据集上框得准，不代表真实夹爪一定能抓起来。真实环境里还会受到深度误差、机器人标定、夹爪摩擦、物体材质、运动规划等因素影响。

## 13. 2D Planar Grasping 的几种范式

2D 抓取方法大致可以分成几类。

| 范式 | 思路 | 代表 |
|---|---|---|
| Direct regression | 直接预测抓取框或 dense grasp map | GG-CNN、GR-ConvNet、NBMOD |
| Grasp evaluation | 先生成候选，再评估质量 | DexNet 2.0 |
| Multi-task | 同时做抓取、分割、refinement | Grasp-Det-Seg |
| Affordance / heatmap | 先预测哪里适合抓 | grasp affordance 方法 |

从更抽象的角度看，2D grasping 其实经常包含两个步骤：

```text
生成候选抓取
评价候选抓取
```

有些方法把这两步合在一个网络里，有些方法显式分开做。

---

## 14. 方法之间怎么理解

几个方法可以这样对比：

| 方法 | 类型 | 简要理解 |
|---|---|---|
| GG-CNN | dense regression | 在每个像素预测抓取质量、角度和宽度 |
| GR-ConvNet | direct regression | 用残差卷积网络直接预测抓取图 |
| NBMOD | detection-like | 把抓取框预测做成类似检测任务 |
| Grasp-Det-Seg | multi-task | 结合检测、分割和 refinement |
| DexNet 2.0 | evaluation | 学习候选抓取的质量评分 |

如果只想先建立直觉，可以这样记：

```text
GG-CNN / GR-ConvNet：从图像直接生成抓取
NBMOD：像检测一样预测抓取框
Grasp-Det-Seg：把抓取和分割结合
DexNet 2.0：先采样候选，再判断好不好
```

## 15. 和前面课程的关系

2D grasping 看起来是一个视觉学习任务，但真正落到机器人上，还需要前面学过的知识。

例如：

- 位姿估计：知道物体在空间中的位置
- 坐标变换：把图像点转到机器人坐标系
- 预抓取位姿：让夹爪先到安全接近位置
- 路径规划：让机械臂避开障碍物
- 轨迹生成：让执行过程平滑可靠
- 控制算法：让机器人按轨迹稳定运动

所以 2D planar grasping 只是完整抓取系统中的一个模块。

最终系统通常是：

```text
图像输入 -> 抓取预测 -> 3D 投影 -> 坐标转换 -> 预抓取 -> 路径规划 -> 轨迹执行
```
