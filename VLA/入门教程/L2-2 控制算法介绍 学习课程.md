# L2-2 控制算法介绍：新手学习课程

---

## 0. 这节课在讲什么

这一节讲的是机器人控制算法。

上一节“规划方法”更关注：

```text
机器人应该走哪条路？
```

这一节“控制算法”更关注：

```text
机器人怎样稳定、准确、平滑地按照目标运动？
```

以理疗按摩机器人为例，机器人不仅要移动到人体表面，还要保持合适的接触力。

例如：

```text
目标：机器人末端以恒定力接触人体
传感器：测量当前接触力
控制器：根据力误差调整机器人位置
系统：机器人本体和环境
```

---

## 1. 控制系统的基本概念

### 1.1 什么是控制

控制就是让系统输出尽量接近目标。

例如按摩机器人希望接触力是 10N：

```text
期望力 r = 10N
当前测得力 y = 8N
误差 e = r - y = 2N
```

控制器看到误差后，会调整机器人位置，让接触力变大。

### 1.2 控制系统的组成

一个基本控制系统包括：

| 模块 | 作用 | 例子 |
|---|---|---|
| Reference | 参考目标 | 期望力、期望位置、期望速度 |
| Sensor | 传感器 | 力传感器、编码器、相机 |
| Controller | 控制器 | PID、MPC |
| Plant/System | 被控系统 | 机器人、机械臂、电机、环境 |
| Output | 系统输出 | 实际位置、实际速度、实际力 |

可以用一句话理解：

```text
传感器告诉控制器现在怎么样，控制器决定下一步怎么调，系统执行后产生新的结果。
```

### 1.3 开环控制和闭环控制

| 类型 | 是否使用反馈 | 特点 |
|---|---|---|
| Open loop 开环 | 不使用反馈 | 简单，但误差无法自动修正 |
| Closed loop 闭环 | 使用反馈 | 能根据误差自动调整 |

开环控制例子：

```text
电机转 3 秒后停止，不管实际转到了哪里。
```

闭环控制例子：

```text
电机不断读取当前位置，没到目标就继续调，到目标附近就减小控制量。
```

机器人中大多数精确控制都依赖闭环反馈。

## 2. 用按摩机器人理解控制系统

这里用力控按摩作为例子。

任务可能有两种：

1. 机器人以恒定期望力接触人体
2. 机器人沿 `x-y` 平面轨迹运动，同时在 `z` 轴保持变化的接触力

### 2.1 控制问题怎么写

假设：

```text
r = desired force，期望力
y = measured force，当前测得力
e = r - y，力误差
u = position displacement，控制器输出的位置调整量
```

控制系统做的是：

```text
期望力 r
    |
    v
与当前力 y 比较，得到误差 e
    |
    v
控制器根据 e 计算位置调整量 u
    |
    v
机器人上下移动
    |
    v
接触力 y 发生变化
    |
    v
传感器再次测量 y，形成闭环
```

新手要抓住重点：

```text
力控不一定直接输出力，也可以通过调整位置来间接改变力。
```

## 3. 系统建模：控制器为什么需要模型

### 3.1 什么是系统模型

系统模型描述：

```text
输入怎么影响输出
```

例如：

```text
给电机一个电压，电机速度怎么变化？
机械臂末端向下移动 1mm，接触力会增加多少？
```

控制器越了解系统，越能做出合适的控制决策。

### 3.2 微分方程

很多物理系统可以用微分方程描述。

例如质量-弹簧-阻尼系统：

```text
M x'' + D x' + K x = F
```

这里：

| 符号 | 含义 |
|---|---|
| `M` | 惯性，质量特性 |
| `D` | 阻尼，速度相关阻力 |
| `K` | 刚度，位置偏移产生的恢复力 |
| `x` | 位置 |
| `x'` | 速度 |
| `x''` | 加速度 |
| `F` | 外力 |

新手理解：

```text
微分方程是在描述系统随时间怎么变化。
```

### 3.3 传递函数

传递函数通常写成：

```text
G(s) = Y(s) / X(s)
```

意思是：

```text
输出 / 输入
```

它来自微分方程经过拉普拉斯变换后的表达。

新手不必一开始就推导公式，先记住：

```text
传递函数用来描述一个线性系统输入和输出之间的动态关系。
```

### 3.4 状态空间

状态空间形式常写成：

```text
x_dot = A x + B u
y     = C x + D u
```

其中：

| 符号 | 含义 |
|---|---|
| `x` | 系统状态 |
| `u` | 控制输入 |
| `y` | 系统输出 |
| `A` | 状态如何自己变化 |
| `B` | 输入如何影响状态 |
| `C` | 状态如何变成输出 |
| `D` | 输入如何直接影响输出 |

例子：

```text
x 可以包含位置和速度
u 可以是电机力矩或期望速度
y 可以是实际测量到的位置或力
```

状态空间是 MPC 等现代控制方法常用的表达方式。

## 4. Block Diagram：控制框图怎么读

控制框图是一种图形化表达方式。

常见元素包括：

| 元素 | 含义 |
|---|---|
| 方块 | 一个系统、控制器或传递函数 |
| 箭头 | 信号流动方向 |
| 加减号 | 信号相加或相减 |
| 反馈线 | 把输出测量值送回输入侧 |

最典型闭环结构：

```text
参考目标 r -> 比较器 -> 误差 e -> 控制器 -> 系统 -> 输出 y
                   ^                              |
                   |                              |
                   +---------- 传感器反馈 --------+
```

读控制框图时，先找三件事：

1. 目标输入是什么
2. 输出是什么
3. 反馈信号怎么回到控制器

## 5. PID 控制器

PID 是最经典、最常用的控制器之一。

PID 包括三个部分：

| 项 | 名称 | 作用 |
|---|---|---|
| P | Proportional，比例 | 当前误差越大，控制量越大 |
| I | Integral，积分 | 消除长期残留误差 |
| D | Derivative，微分 | 抑制变化过快，增加阻尼 |

PID 控制律可以理解为：

```text
控制量 = Kp * 当前误差 + Ki * 误差累积 + Kd * 误差变化速度
```

### 5.1 P 控制

P 控制只看当前误差。

```text
误差大 -> 用力调
误差小 -> 轻轻调
```

问题是：只用 P 控制可能存在稳态误差。

### 5.2 I 控制

I 控制看误差的累积。

如果系统长期差一点到不了目标，积分项会不断累积，推动系统继续调整。

优点：

```text
可以消除稳态误差
```

缺点：

```text
可能降低稳定性
可能出现积分饱和
```

所以实践中积分项常常需要 clip，也就是限制最大值和最小值。

### 5.3 D 控制

D 控制看误差变化速度。

如果系统正在快速接近目标，D 项会起到刹车作用，减少 overshoot。

这里要注意：

```text
PD 控制可以增加 damping ratio，减少 overshoot，但容易受噪声影响。
```

这里的意思是：

```text
D 项能让系统更稳，但传感器噪声会被微分放大。
```

所以实际使用 D 项时，经常需要滤波。

### 5.4 damping ratio 和 overshoot

| 术语 | 新手理解 |
|---|---|
| damping ratio 阻尼比 | 系统有多“稳”、多不容易来回振 |
| overshoot 超调 | 目标是 10，结果冲到了 12 |
| steady-state error 稳态误差 | 最后稳定下来后还差一点 |

调 PID 时经常关注：

```text
响应够不够快？
有没有超调？
最后误差能不能消掉？
会不会振荡？
```

### 5.5 PID 实践注意事项

PID 调参有几个非常实用的点：

1. 关注传感器噪声
2. 必要时滤波
3. 积分项最好限制，防止误差非预期累积
4. 注意硬件能力限制

例如机器人电机有最大速度、最大力矩、最大加速度，如果控制器输出超过硬件能力，系统可能无法执行，甚至产生危险。

## 6. PID 参数怎么调

一种经典调参方法是 Ziegler-Nichols。

新手不需要马上掌握完整公式，先理解调参方向。

### 6.1 参数的直觉影响

| 参数 | 增大后可能的效果 | 风险 |
|---|---|---|
| `Kp` | 响应更快 | 超调、振荡 |
| `Ki` | 稳态误差变小 | 积分饱和、变慢、不稳定 |
| `Kd` | 减少超调、增加阻尼 | 对噪声敏感 |

### 6.2 一个保守的调参思路

可以按这个顺序理解：

```text
先调 Kp，让系统能较快接近目标
再加少量 Ki，消除稳态误差
最后加 Kd，减少超调和振荡
```

真实机器人上调参要非常小心，通常从小增益开始，逐步增加。

## 7. 控制系统稳定性分析

控制理论里有几个关键词：

1. pole，极点
2. zero，零点
3. characteristic equation，特征方程
4. root locus，根轨迹
5. LHP，左半平面

### 7.1 极点和稳定性

对连续时间线性系统来说，一个常见结论是：

```text
系统稳定要求极点在复平面的左半平面 LHP
```

也就是极点实部要小于 0。

新手先这样记：

```text
极点位置决定系统响应会不会发散、振荡、衰减。
```

### 7.2 根轨迹

根轨迹展示的是：

```text
当控制增益 K 从 0 变化到无穷大时，闭环系统极点在复平面上怎么移动。
```

它的作用是帮助我们看：

1. 增益变大后系统是否稳定
2. 系统响应会变快还是变慢
3. 是否可能出现振荡或超调

## 8. MPC 控制器

MPC 全称是 Model Predictive Control，模型预测控制。

### 8.1 MPC 的核心思想

MPC 会用模型预测未来一段时间系统怎么变化，然后求解一个优化问题。

可以理解为：

```text
先预测未来多步
再选择当前最合适的一步控制
执行一步后重新测量
再重新预测和优化
```

这叫 receding horizon，滚动时域。

### 8.2 MPC 优化什么

MPC 的目标是：

```text
Minimize error with reference
Regulate control input
```

也就是：

1. 尽量贴近参考轨迹或参考目标
2. 控制输入不要太大、太剧烈

一个常见目标函数直觉是：

```text
总成本 = 跟踪误差成本 + 控制输入成本
```

### 8.3 MPC 为什么强

MPC 的优势是能比较自然地处理约束。

例如：

1. 位置边界
2. 速度限制
3. 加速度限制
4. 力矩限制
5. 工作空间限制
6. 避免奇异位形

这对机器人非常重要，因为机器人不是想怎么动就能怎么动。

### 8.4 MPC 的流程

MPC 的基本流程可以整理为：

```text
1. 建立系统模型或状态空间模型
2. 构建优化目标
3. 构建约束
4. 优化求解器求解未来多步控制量
5. 只执行第一步控制
6. 获取新的系统状态
7. 重复以上过程
```

### 8.5 MPC 的应用

典型例子包括：

| 应用 | 说明 |
|---|---|
| 车辆轨迹跟踪 | 根据车辆模型预测未来轨迹，输出在线控制策略 |
| 机器人轨迹优化 | 根据机器人运动学模型和约束，优化末端轨迹 |

新手要记住：

```text
PID 更像根据当前误差直接调。
MPC 更像带着模型和约束，往未来看几步再决定当前动作。
```

## 9. 力控：为什么机器人要控制力

很多机器人任务不只要求“到达位置”，还要求“接触得合适”。

例如：

1. 按摩
2. 打磨
3. 插孔装配
4. 抛光
5. 人机交互

这些任务中，只控制位置可能不够。

例如：

```text
末端位置只差 1mm，但如果接触对象很硬，力可能突然变得很大。
```

所以机器人需要力控。

## 10. 导纳控制

导纳控制英文是 admittance control。

### 10.1 核心思想

导纳控制根据外力调整位置。

导纳控制可以写成：

```text
根据期望力调整位置
通过位置控制来实现追踪期望力
```

新手理解：

```text
测到力太大，就让机器人退一点；
测到力太小，就让机器人压一点。
```

### 10.2 导纳模型

核心模型是：

```text
Md e'' + Dd e' + Kd e = Fext
```

这就是质量-阻尼-弹簧模型。

| 符号 | 含义 |
|---|---|
| `Md` | 期望惯性 |
| `Dd` | 期望阻尼 |
| `Kd` | 期望刚度 |
| `e` | 位置误差或位移关系 |
| `Fext` | 外部接触力 |

导纳控制做的事情是：

```text
外力 Fext -> 通过质量-阻尼-弹簧模型 -> 得到期望位置 xd
```

然后再用位置控制器，比如 PD 控制器，让机器人追踪这个 `xd`。

### 10.3 导纳控制实现方式

这里要注意：

1. 可以通过关节力矩传感器估计末端力
2. 也可以通过末端力传感器直接测量

一般来说：

```text
末端力传感器成本更高，但可靠性更好。
```

### 10.4 导纳控制适合什么场景

导纳控制通常更柔顺，适合和柔性环境、人接触的任务。

例如：

```text
按摩机器人
人机交互
柔性物体接触
```

## 11. 阻抗控制

阻抗控制英文是 impedance control。

### 11.1 核心思想

阻抗控制直接控制力或力矩，使机器人表现出期望的质量-阻尼-弹簧关系。

阻抗控制的核心思想是：

```text
阻抗控制直接控制力矩来达到期望力。
```

新手理解：

```text
阻抗控制更直接地管机器人输出的力。
```

### 11.2 阻抗控制和导纳控制的共同点

两者都基于质量-阻尼-弹簧系统：

```text
Md e'' + Dd e' + Kd e = Fext
```

它们都在描述：

```text
位置变化和接触力之间应该满足什么关系。
```

### 11.3 阻抗控制 vs 导纳控制

二者的区别可以整理为：

| 对比 | 阻抗控制 | 导纳控制 |
|---|---|---|
| 输出 | 力或力矩 | 位置 |
| 响应和精度 | 通常更好 | 通常更柔顺 |
| 适合环境 | 刚性环境 | 柔性环境 |
| 实现要求 | 对动力学和力矩控制要求更高 | 可通过位置控制实现 |

一句话记忆：

```text
导纳控制：力进来，位置出去。
阻抗控制：位置关系进去，力或力矩出去。
```

## 12. PID、MPC、导纳、阻抗之间的关系

这些控制方法不是互相孤立的。

例如按摩机器人可以这样组合：

```text
外层：导纳控制
    根据接触力计算期望位置

内层：PD / PID 控制
    让机器人追踪期望位置
```

也可以这样：

```text
MPC
    在考虑速度、力、工作空间等约束的情况下优化未来运动
```

所以：

| 方法 | 更像在解决 |
|---|---|
| PID | 当前误差怎么快速修正 |
| MPC | 有模型和约束时，未来几步怎么优化 |
| 导纳控制 | 外力变化时，位置应该怎么顺应 |
| 阻抗控制 | 如何直接塑造力和运动的动态关系 |
